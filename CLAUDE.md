# CPSC 430 — Computers and Society (UBC, 2026 Summer 2)

Personal coursework directory for an ethics/writing course on the social and ethical
implications of computing. This is **not a code project** — the work here is *writing*
(argumentative mini-essays), reading, and lecture note-taking. There is nothing to build,
run, or test.

## Layout

| Directory | Contents |
|---|---|
| `admin/` | Syllabus and course logistics (`syllabus.pdf`) |
| `lecture/` | Lecture notes in Markdown, `lecture-N.md` |
| `essays/` | One folder per mini-essay: `e1/`, `e2/`, … |
| `reading-materials/` | Course textbook (`textbook.pdf` — Quinn, *Ethics for the Information Age*) |
| `slides/` | Lecture slide decks (`.pptx`) |
| `quizzes/` | Reading-quiz material (currently empty) |

## Essay folders (`essays/eN/`)

Each essay gets its own folder. Typical contents:
- `prompt.md` — the assignment prompt (choose one of N sub-prompts to argue for/against).
- `draft.md` — the student's working draft (may be empty until started).
- Source articles saved as Markdown (e.g. `a1-*.md`, `a2-*.md`) when the prompt is
  reacting to specific readings.

Essay prompts are **argumentative**: pick a position and defend it. Prompts reference
textbook sections (e.g. "Secs. 2.1–2.5", ethical egoism, Quinn's standard for evaluating
ethical theories).

## Course mechanics (from `lecture/lecture-1.md`)

- **10 mini-essays total, ~2 per week**; essay weight increases over the term.
- Essays are written in the **CBTF** (Computer-Based Testing Facility). A ~150-character
  outline is drafted in **MTA** beforehand and carries into the CBTF.
- **Peer review** is central: ≥3 peer reviews per essay; a TA's grade overrides peer
  grades; grades can be appealed (TA regrade is final).
- Must complete the **reading quiz** before doing peer reviews.
- **Calibrated reviews** (grading pre-scored essays) build a "dependability score."

## Note-taking conventions (`lecture/lecture-N.md`)

Lecture notes follow a consistent structure worth preserving when editing/adding:
- `### General Notes` — logistics and content
- `### Small Group Discussion` — in-class debate positions and supporting points
- `### Questions` — `Q:` / `A:` pairs; some answers are left blank to fill in later
- `### Action Items` — deadlines and to-dos

## Working in this directory

- When asked to help with an essay, work in the relevant `essays/eN/draft.md`, following
  the position/sub-prompt in that folder's `prompt.md`.
- PDFs and `.pptx` are binary; use the Read tool's `pages` parameter for the textbook PDF.
- This is graded academic writing — help with structure, argument, and clarity, but the
  student's own reasoning and voice should drive the essays.
